Zynrax provides a comprehensive suite of moderation commands designed for speed and reliability. From basic actions like kicking and banning to bulk role management and channel resets, everything is built to help you maintain order with ease.

## Core Moderation

Essential commands for daily server discipline.

<AccordionGroup>
  <Accordion title="Ban & Unban" icon="gavel">
    Remove members permanently or restore their access.
    <CodeGroup>
      ```bash Syntax
      ,ban (member) [reason]
      ,unban (member) [reason]
      ```
      ```bash Example
      ,ban @Montage Trolling
      ,unban 371434681784205312 Appeal accepted
      ```
    </CodeGroup>
    <Note>
      Use `,unbanall` to clear the entire server ban list at once.
    </Note>
  </Accordion>

  <Accordion title="Kick" icon="user-minus">
    Instantly remove a member from the server.
    <CodeGroup>
      ```bash Syntax
      ,kick (member) [reason]
      ```
      ```bash Example
      ,kick @Montage Spam
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Mute (Timeout)" icon="comment-slash">
    Temporarily restrict a user's ability to speak or interact using Discord's timeout system.
    <CodeGroup>
      ```bash Syntax
      ,mute (member) (duration) [reason]
      ,unmute (member)
      ```
      ```bash Example
      ,mute @Montage 1h Spamming
      ,unmute @Montage
      ```
    </CodeGroup>
    <Info>
      Durations can be `s` (seconds), `m` (minutes), `h` (hours), or `d` (days). Max duration is 28 days.
    </Info>
    <Note>
       Use `,unmuteall` to lift all active timeouts in the server simultaneously.
    </Note>
  </Accordion>

  <Accordion title="Warnings" icon="triangle-exclamation">
    Keep track of member violations. Zynrax automatically applies a **timeout** if a user reaches 4+ warnings.
    <CodeGroup>
      ```bash Syntax
      ,warn (member) [reason]
      ,clearwarns (member)
      ```
      ```bash Example
      ,warn @Montage Bad language
      ,clearwarns @Montage
      ```
    </CodeGroup>
  </Accordion>
</AccordionGroup>

## Message Management

Tools for cleaning chat clutter and auditing deleted content.

### Purge Engine

Delete large amounts of messages (up to 999) with specialized filters.

<AccordionGroup>
  <Accordion title="Basic Purge" icon="broom">
    <CodeGroup>
      ```bash Syntax
      ,purge (amount)
      ```
      ```bash Example
      ,purge 50
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="User Purge" icon="user-gear">
    <CodeGroup>
      ```bash Syntax
      ,purge user @user [amount]
      ```
      ```bash Example
      ,purge user @Montage 100
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Advanced Filters" icon="filter">
    Deep clean specific types of content:
    - `,purge bots`: Deletes messages sent by bots.
    - `,purge images`: Deletes messages containing images.
    - `,purge embeds`: Deletes messages with rich embeds.
    - `,purge files`: Deletes messages with any uploaded file.
    - `,purge emoji`: Deletes messages containing custom emojis.
    - `,purge contains [text]`: Deletes messages containing specific keywords.
    - `,purge reactions [amount]`: Removes all reactions from recent messages.
    
    <CodeGroup>
      ```bash Syntax
      ,purge (filter) [amount]
      ```
      ```bash Example
      ,purge bots 20
      ,purge images 50
      ,purge contains scam
      ```
    </CodeGroup>
  </Accordion>
</AccordionGroup>

### Snipe Commands

View content that was recently removed from the channel.

<CodeGroup>
  ```bash Syntax
  ,snipe
  ```
</CodeGroup>

<Info>
  The `snipe` command retrieves the last deleted message, including its author, content, and any attachments.
</Info>

## Channel Control

Management tools for channel visibility and structure.

<AccordionGroup>
  <Accordion title="Privacy & Chat Control" icon="lock">
    Disable or enable sending messages and visibility in the current channel.
    <CodeGroup>
      ```bash Syntax
      ,lock
      ,unlock
      ,hide
      ,unhide
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Slowmode" icon="clock">
    Limit how often users can send messages.
    <CodeGroup>
      ```bash Syntax
      ,slowmode (seconds)
      ```
      ```bash Example
      ,slowmode 5
      ```
    </CodeGroup>
    <Note>
      Use `,unslowmode` to quickly remove any active slowmode.
    </Note>
  </Accordion>

  <Accordion title="Bulk Channel Actions" icon="layer-group">
    Apply permissions across the entire server instantly.
    <CodeGroup>
      ```bash Syntax
      ,lockall
      ,unlockall
      ,hideall
      ,unhideall
      ```
    </CodeGroup>
    <Important>
      Bulk channel actions require your role to be **strictly above** the bot's top role to prevent unauthorized mass changes.
    </Important>
  </Accordion>

  <Accordion title="Channel Infrastructure" icon="explosion">
    Quickly reset or duplicate channels.
    <CodeGroup>
      ```bash Syntax
      ,nuke
      ,clone (channel)
      ```
      ```bash Example
      ,clone #general
      ```
    </CodeGroup>
  </Accordion>
</AccordionGroup>

## Role Management

Zynrax offers the most powerful role management system, allowing for both individual and bulk assignments.

### Single Role Management
Toggle, create, or modify roles with ease.
<CodeGroup>
  ```bash Syntax
  ,role (member) (role)
  ,role create (name)
  ,role delete (role)
  ,role rename (role) (name)
  ```
  ```bash Example
  ,role @Montage @VIP
  ,role create Moderators
  ,role rename @VIP Premium
  ```
</CodeGroup>

### Bulk Role Operations
Assign or remove roles from entire groups of users at once.

<AccordionGroup>
  <Accordion title="Bulk Add" icon="user-plus">
    Give a role to every member fitting a specific criteria.
    <CodeGroup>
      ```bash Syntax
      ,role all (role)
      ,role humans (role)
      ,role bots (role)
      ,role unverified (role)
      ```
      ```bash Example
      ,role all @Member
      ,role humans @Verified
      ```
    </CodeGroup>
  </Accordion>

  <Accordion title="Bulk Remove" icon="user-minus">
    Remove a role from every member fitting a specific criteria.
    <CodeGroup>
      ```bash Syntax
      ,removerole all (role)
      ,removerole humans (role)
      ,removerole bots (role)
      ```
      ```bash Example
      ,removerole all @Muted
      ,removerole bots @Verified
      ```
    </CodeGroup>
  </Accordion>
</AccordionGroup>

<Warning>
  - Roles with **Dangerous Permissions** cannot be managed via bulk commands. These include:
    `Administrator`, `Manage Server`, `Manage Roles`, `Manage Channels`, `Kick Members`, `Ban Members`, `Manage Webhooks`, and `Mention @everyone`.
  - To use **Bulk Add/Remove** commands, your top role must be **higher than or equal to** the bot's top role.
</Warning>
